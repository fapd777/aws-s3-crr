⏺ Working end-to-end. Here's what the problem was and what was changed:

  Root cause: The bundled terraform plugin in tflint 0.60.0 predates v0.14.0 of tflint-ruleset-terraform, which is
  the version that added unused provider alias detection to the terraform_unused_declarations rule.

  Fix — .tflint.hcl now pins the external plugin explicitly:

  plugin "terraform" {
      enabled = true
      version = "0.14.0"
      source  = "github.com/terraform-linters/tflint-ruleset-terraform"
      preset  = "recommended"
  }

  Adding source + version forces tflint --init to download the specific external release instead of falling back to
  the bundled (older) copy.

  Sources:
  - terraform_unused_declarations rule docs
  - Discussion: provider check configurability (v0.14.0)
